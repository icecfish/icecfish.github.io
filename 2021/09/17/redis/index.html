Redis字符串（String）

简介

String类型是二进制安全的，意味着Redis的stirng可以包含任何数据，比如jpg图片或者序列化的对象。

String类型是Redis中最基本的数据类型，一个Redis中字符串value最多可以存放512M

补充：原子操作

incr/decr/incrby/decrby是原子性操作

原子操作：不会被线程调度机制打断的操作。这种操作一旦开始就一直运行到结束，中间不会有任何的 context swtich（切换到另一个线程）

在单线程中，能够在单条指令中完成的操作都认为是原子操作，因为中断了之恩那个发生于指令之间。

在多线程中，不能被其他进程打断的操作就叫做原子操作。

Redis列表(List)

简介

Redis列表是简单的字符串列表。底层实际上是一个双向链表，对两端的操作性高,查询效率低。一键多值。

常用命令

补充：数据结构

List的数据结构为快速链表quicklist

在元素较少的情况下会使用一块连续存储的内存存储，这个结构是ziplist，也即是压缩列表。将所有的元素紧挨着放在一起存储，分区的是一块连续的内存。

当数据量比较多的时候才会改成quicklist，将多个压缩链表用链表的方式连起来构成快速链表。

Redis集合(set)

简介

Redis set对外提供的功能与list类似是一个列表功能，但是set可以自动排重。

Redis set是String类型的无序集合。它底层其实是一个value为null的hash表，所以添加，删除，查找的复杂度都是O(1)

常用命令

补充：数据结构

使用hash结构，所有的value指向同一个value值。类似java中的hashset

Redis哈希(Hash)

简介

Redis hash是一个键值对集合

Redis hash是一个string类型的filed和value的映射表，hash特别适合用于存储对象，类似Java中的Map<String,Object>。hash里value值还是hash

常用命令

补充：数据结构

hash类型对应的数据结构是两种：ziplist,hashtable.当field-value长度较短且个数较少时，使用ziplist，否则使用hashtable。
